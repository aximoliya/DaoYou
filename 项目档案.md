## 项目简介

**道友攻略站**是一个面向手游《道友请留步》玩家的静态攻略网站。

该网站将所有分散在社区（TapTap、贴吧等）中的零散攻略内容系统化整理，以结构化的网页形式呈现，帮助玩家快速找到开箱路线、商店购买优先级和养成系统的最佳策略。

核心理念：**把"攻略"从聊天截图和帖子长文变成可以搜索、可筛选、层级清晰的在线文档**。

---

## 项目目标

1. **知识沉淀**：将知名攻略作者"笑语"的完整攻略内容结构化存储，避免丢失
2. **高效检索**：提供全文搜索 + 品质等级筛选，让玩家秒找到想要的开箱路线
3. **持续更新**：便于维护者不断追加新版本攻略，保证时效性
4. **移动友好**：玩家能在手机上边玩游戏边查攻略
5. **零运营成本**：纯静态站，托管在 GitHub Pages，无服务器费用

## 数据结构说明

### 导航数据 (src/data/navItems.ts)

```typescript
interface NavItem {
  href: string;      // 路由路径，如 '/kaixiang/shenxian/'
  label: string;     // 显示文本，如 '一、神仙'
  group?: string;    // 分组标题，如 '开箱攻略'（可选）
}
```

四个预设导航数组：
- `globalNavItems` - 全站17个链接（首页侧边栏使用）
- `kaixiangNavItems` - 开箱攻略11个链接
- `richangNavItems` - 日常运营3个链接
- `yangchengNavItems` - 养成系统2个链接

### 品质等级标识

`data-tiers` 属性附着在 HTML 元素上，表示该内容属于哪些品质等级：

| 属性值 | 含义 |
|--------|------|
| `xh` | 曦幻（最高品质） |
| `ll` | 琉璃 |
| `jx` | 金霄 |
| `ct` | 赤魇 |
| `zx` | 紫虚 |
| `qy` | 青曜 |
| `hc` | 幻彩 |

TierBar 筛选器通过匹配 `data-tiers` 属性来显隐内容块。

### 页面元数据

通过 `data-page` 和 `data-current` 属性挂在 `<html>` 上：
- `data-page`: 当前所在分类（kaixiang / richang / yangcheng / hub）
- `data-current`: 当前具体页面（shenxian / shop / shencang 等）

## 搜索功能说明

**完全纯前端、实时、无依赖**：

1. 用户在 `#search` 输入框中输入关键词
2. `search.ts` 监听 input 事件（300ms 防抖）
3. 遍历页面所有 `<section>` 元素，检查 `textContent` 是否包含关键词（不区分大小写）
4. 不匹配的 section 被添加 `.hidden` 类（display:none）
5. 对可见 section 内的 h2/h3/.card/p/td/li/.flow/.info/.warn/.card-title 元素进行 TreeWalker 文本节点遍历
6. 匹配到的文本用 `<mark class="search-hit">` 包裹高亮
7. 当前高亮结果用红色标记（`.current`），其余用紫色
8. 上/下导航按钮循环浏览所有匹配项，自动滚动到视口中央
9. 搜索与品质筛选联动：TierBar 变化时触发搜索重跑（`tier-changed` 事件）

## 已知问题

1. **内容硬编码**：所有攻略内容写在 `.astro` 文件的 HTML 中，修改内容需要动源码
2. **两个 navItems.ts**：`src/data/navItems.ts` 和 `src/lib/navItems.ts` 存在重复的 NavItem 接口定义
3. **tianshu.astro 有错误过滤**：第27行 `navItems.filter(item => item.href !== 'duoxuan.html')` 应该是 `.html` 的后缀错误 - 其余都是 `/kaixiang/xxx/` 格式
4. **BackToTop 组件重复创建**：`BackToTop.astro` 渲染了一个静态按钮，`back-to-top.ts` 又动态创建了一个（通过 id 检查避免重复，但设计冗余）
5. **TOCPanel 空占位**：组件只有空的 div，实际内容完全由 JS 动态注入
6. **部分页面无搜索/TierBar**：`richang/huodong.astro` 和分类总览页缺少搜索和 TierBar，不一致
7. **huodong.astro 开头有 BOM 字符**：`﻿---`（UTF-8 BOM），可能导致解析问题
8. **PWA 图标路径**：manifest.json 引用 `/pwa-192x192.png` 和 `/pwa-512x512.png`，但 public 目录中没有这些文件
9. **og-image.png 缺失**：BaseLayout 引用 `/og-image.png` 但不存在
10. **没有路由级 404 页面**：不存在 `src/pages/404.astro`

## 注意事项

1. **base 路径**：所有链接必须带 `BASE_URL`（`/DaoYou/`），否则 GitHub Pages 部署会 404
2. **路由格式**：Astro 文件路由，`/kaixiang/shenxian.astro` → `/kaixiang/shenxian/`
3. **Node 版本**：GitHub Actions 使用 Node 20 构建
4. **输出模式**：`output: 'static'` 纯静态，SSR 相关功能不可用
5. **攻略版权**：内容来自"笑语@TapTap"，需保持来源标注
6. **CSS 变量**：所有颜色通过 CSS 变量控制，修改设计系统只需改 `_variables.css`
